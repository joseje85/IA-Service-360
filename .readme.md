# 🚀 Configuración de Producción - API IA (FastAPI + IIS + NSSM)

Este documento describe la configuración completa del servidor para ejecutar el servicio de IA en producción.

---

# 🧩 1. Arquitectura General

```
Cliente (Red)
   ↓
IIS (Puerto 81 / 80 / 443)
   ↓
Uvicorn (Servicio NSSM - puerto 8001)
   ↓
FastAPI
   ↓
Ollama (puerto 11434)
```

---

# ⚙️ 2. IIS (Internet Information Services)

## 📍 Ubicación del sitio

```
C:\inetpub\wwwroot\api-ia
```

## 📄 web.config

```xml
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="ReverseProxyInboundRule" stopProcessing="true">
          <match url="(.*)" />
          <action type="Rewrite" url="http://127.0.0.1:8001/{R:1}" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
```

## 🔗 Bindings

Configuración recomendada:

```
Type: http
IP Address: All Unassigned
Port: 81
```

# 🔥 5. Firewall (Windows Defender)

## 🚨 Problema detectado

El firewall bloqueaba el acceso desde otras máquinas.

---

## ✅ Solución aplicada

### Permitir ICMP (ping)

```bash
netsh advfirewall firewall add rule name="Allow ICMP V4" protocol=icmpv4 dir=in action=allow
```

---

### Permitir puerto IIS (81)

```bash
netsh advfirewall firewall add rule name="IIS 81" dir=in action=allow protocol=TCP localport=81
```

---

### Regla completa (recomendada)

```bash
netsh advfirewall firewall add rule name="IIS 81 FULL" dir=in action=allow protocol=TCP localport=81 profile=any
```

---

## 🔄 Activar / Desactivar firewall

```bash
# Desactivar (solo pruebas)
netsh advfirewall set allprofiles state off

# Activar nuevamente
netsh advfirewall set allprofiles state on
```

# 🧠 9. Notas importantes

* No usar `--reload` en producción
* Usar `127.0.0.1` en lugar de `localhost`
* Mantener firewall activo con reglas específicas
* IIS solo actúa como proxy
* FastAPI corre en segundo plano (NSSM)
